PDP 자동화 카테고리 분류 기준 v3

마이리얼트립 offer-categories + offer-sub-categories + 실제 PDP 이미지 분석(티켓 13개 + 투어 5개 + 디자인 가이드) 기반으로
4개 PDP 자동화 카테고리를 정의합니다. ① 티켓·이용권은 5개 서브타입으로 세분화.

분류 기준: PDP에서 공통으로 필요한 섹션 타입이 70% 이상 겹치면 같은 카테고리로 통합.
핵심 질문 — "이 상품의 PDP를 만들 때 가장 중요한 정보 타입이 무엇인가?"

분류 플로우

Input — 데이터 소스
📋offer-categories (기존 37개 카테고리)
📂offer-sub-categories (서브 카테고리)
📸실제 PDP 스크린샷 18개 분석
🎨T&A 디자인 가이드 이미지

🔍 분류 기준 (3가지 축)

  • ① 핵심 정보 타입 (티켓 vs 코스 vs 패스 vs 서비스)
  • ② PDP 필수 섹션 70% 겹침 여부
  • ③ 구매 결정 요인 유사성 (가격비교 vs 신뢰 vs 편의)
Output — 4개 PDP 자동화 카테고리 (① 티켓·이용권은 5개 서브타입 포함)
①
티켓·이용권
현장 QR/실물 제시로 이용하는 모든 티켓·이용권 상품. 5개 서브타입으로 세분화.
1A · 테마파크 입장권 — 티켓 옵션표 + QR 입장
1B · 교통 티켓 — 구간 배지 + 노선도
1C · 시티패스·멀티패스 — 포함 시설 + 절약 계산
1D · 유람선·크루즈·공연 — 시간표 + 탑승 방법
1E · 스냅·스파·미식 이용권 — 업소 목록 + 서비스 안내
②
가이드 투어
핵심: 가이드 신뢰 + 코스 타임라인
바티칸 투어 루브르 시드니 선셋 발리 지프
③
골프·특수투어
핵심: 코스 스펙 표 + 픽업 + 특수 주의
골프 투어 수향마을 오로라 투어
④
세미패키지
핵심: 호텔 숙박 + Day별 일정 + 1인 추가요금
디즈니 2박 3일 오사카 3박 4일 발리 리조트 패키지

신규 상품 카테고리 판별 로직

❓ 현장 QR/바우처 제시로 이용하는 티켓·이용권인가?
→ ① 티켓·이용권 — 서브타입을 추가 선택:
· 날짜/종류별 입장권 → 1A 테마파크 · 특정 구간 이동 교통편 → 1B 교통 티켓 · 여러 시설/노선 하나의 패스 → 1C 시티패스 · 시간표 있는 유람선/공연 → 1D 유람선·공연 · 스냅/스파/미식 업소 이용권 → 1E 스냅·스파·미식
❓ 가이드와 함께 특정 코스를 도는 투어인가?
→ ② 가이드 투어
경력 배지 + 코스 타임라인 + 집합 안내 구조 적용
❓ 코스/장소별 스펙 비교 + 픽업이 필요한가?
→ ③ 골프·특수투어
스펙 비교표 + 픽업 + 특수 주의사항 구조 적용
❓ 호텔 숙박이 포함된 N박 N일 상품인가?
→ ④ 세미패키지
숙박 안내 + Day별 타임라인 + 1인 추가요금(싱글 서플리먼트) + 이중 취소정책 구조 적용
⚠️ 위 5개 중 어디도 맞지 않는 경우
→ TF 협의 후 카테고리 확장 or 가장 유사한 카테고리 임시 적용
v3에서 카테고리 추가 검토
⚠ v2 적용 시 주의사항
본 카테고리 분류는 v3이며, TF 실제 개발 적용 전 경험 기반팀(Experience팀) 검토가 필요합니다.
① 티켓·이용권 내 서브타입(1A~1E)은 파트너 상품 등록 시 2단계 선택으로 처리됩니다 (카테고리 → 서브타입).
복합형 상품 (예: 골프 투어 + 가이드 포함)은 핵심 섹션 기준으로 가장 비중 높은 카테고리 선택.
